Bro: the syntax-highlighting pager and viewer

Bro is the display dog: its purview is rendering and it owns no repo data — its state is just an in-memory view stack. It renders syntax-highlighted code, Spot results, and Graf diffs in an interactive TUI pager. The goal is one viewer for every dog's output, driven by URIs. The method: take URIs as arguments or TLV hunks on stdin (pipe mode), color text from tok32 tags, keep a 32-deep view stack, and expose a :/# GURI prompt so any position is a re-openable URI.

  • Hunks arrive as TLV records with a path#symbol:line URI; the display title is formatted at render time from that URI.
  • Text is colored from tok32 tag arrays: comment gray, string green, number cyan, keyword blue, name/call bold, path violet.
  • A U tag is an invisible click target — a left-click on the preceding token navigates to its URI.
  • Opening a file or dir pushes the current view (max 32 deep); each stacked view keeps its scroll, h pops, q quits.
  • Long lines soft-wrap by codepoint; w toggles wrap off (less -S clipping), m toggles mouse wheel/click.

API

Bro follows the Dog three-call contract; above it sit the run loops, the tokenizer, and the SGR mouse helpers.

  • BROOpen / BROClose — set up the pager state and arena, then tear it down.
  • BROExec(state, cli) — entry for a parsed CLI of URIs and flags.
  • BRORun — the interactive pager loop (key handling, render, view stack).
  • BROPipeRun — pipe mode: read TLV hunks from stdin and page them.
  • BROUpdate — recompute the rendered view after a navigation or resize.
  • BROTokenize — tokenize a file's bytes into colored tok32 spans.
  • BROListDir — render a directory listing as a view.
  • MAUSEnable / MAUSDisable / MAUSParse — SGR mouse tracking on/off and event parsing.

CLI

bro [URI...] views files, directories, or piped hunks; spot and graf fork it automatically, so it is rarely invoked by hand.

  • bro file.c is highlighted cat; bro file.c#42 opens at line 42; bro file.c#Token opens and searches for a token.
  • bro dir/ or bro . lists a directory; multiple path args view several files; no args at a tty prints the key map.
  • No args + piped stdin = pipe mode (TLV from spot/graf); spot -g TODO .c and graf -d a b fork bro for their output.

The URI prompt

The : (or #) prompt accepts GURI syntax, and the fragment picks the search type; Tab completes identifiers from the current view.

  • :42 goes to a line, :path opens a file, :path#token searches, :#TODO.c greps, :#'snippet'.c snippet-searches.
  • A trailing .ext filters results by type; when omitted, the current file's extension is the default.
  • Tab tries a prefix match then a substring match over identifiers in the current view.

Navigation and status

Keys cover line/page/hunk motion and the view stack; the status bar shows a live, re-typeable URI of the current position.

  • Keys: j/k lines, d/u half-pages, space/f/b pages, g/G ends, ]/[ hunks, Enter/l open, h back, q quit.
  • The status bar's left is <path>#L<line>, updating live as you scroll, so : re-opens the prompt where you are.
  • Position shows TOP/BOT/NN%/ALL; H is hunk count and C change count (diffs only).
