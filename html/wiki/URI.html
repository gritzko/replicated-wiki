URI: scheme://authority/path?ref#fragment

Every be argument is one URI — [scheme:][//authority][path][?ref][#fragment] — and each component reflects one aspect of the command. The verb picks the direction; the URI picks the resource. Changing the URI's shape changes the command's semantics: the same PATCH becomes a merge, a rebase, or a cherry-pick depending on which slots are filled.

  • scheme: is the transport (ssh:/https:/file:) or a read-only projector (sha1:/diff:/…); none acts locally.
  • //authority is a remote label; bare //host reads the cache (no wire, reflog grep), a transport scheme opens the wire.
  • path is a file or dir in the branch's tree; with file: it is a filesystem path to another store/wt on this host.
  • ?ref is a branch, tag, sha, or commit-message search; the ref shape alone decides which (see Refs).
  • #fragment is free-form payload: a commit-message body, an object hash, a line jump, or an extension filter (#.c.h).

Shape changes semantics

Each populated URI slot adds its effect orthogonally, and the slot combination determines the operation — most visibly for PATCH, where the ?/# shape alone selects the merge mode.

  • be patch ?feat squashes, ?feat#msg merges, ?feat# rebases one, #sha cherry-picks — one PATCH verb, varied shape.
  • A bare //host reads the cache while ssh://host opens the wire — the scheme alone flips cached vs network.
  • diff: vs diff:?br vs diff:?a#b change the diff from/to sides — projector slots compose like verb slots.
  • An empty ? slot in a URI means the project trunk; no ? slot at all means cur (see Refs rule 0).

Argument classification

Every non-flag argv token becomes one URI via DOGNormalizeArg; the token's markers decide the slot, so the common case needs no punctuation.

  • An argv token with whitespace is a fragment (be post 'fix the typo'); ?/#// or a scheme: send it to the URI lexer.
  • A bareword argv token with no markers lands in the verb's natural slot via DOGPromoteBareword (see Verbs).
  • One scheme per URI: sha1: is itself the scheme, so sha1:ssh://… is not a valid shape — fetch then read separately.
