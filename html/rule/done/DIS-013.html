DIS-013: mark --strict over a directory never exits non-zero

Mark documents mark --strict wiki/ as "build a dir, fail on any budget breach" and "--strict turns the first breach into a non-zero exit for CI", but the directory walk swallows every per-file error and returns OK, so the documented CI gate always exits 0 — budget breaches across a wiki go undetected. Single-file mode is correct (exit 157/MARKLIMIT). The goal is a directory --strict that fails the build. See Mark.

Issues

The per-file callback discards the render result.

  • mark/MARK.cli.c:89-100 (mark_dir_cb): line ~98 logs the error then return OK;, so FILEScanFiles sees only successes and the process exits 0.
  • Repro: a directory containing a two-H1 page run under --strict prints "failed: MARKLIMIT" yet exits 0; the same page as a single-file arg exits 157.

Blockers

None.

Planned

Repro first, then propagate the first breach out of the walk.

  • Make mark_dir_cb remember the first non-OK under --strict and return it (or set a flag the CLI converts to a non-zero exit after the walk).
  • Add a table case asserting a non-zero exit for a directory with a budget-breaching page under --strict.
