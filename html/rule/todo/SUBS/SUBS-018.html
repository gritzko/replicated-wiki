SUBS-018: a path-scoped be get <path> recurses every submodule unscoped, re-checking-out the whole sub

be get --force graf/DAG.c (a single tracked file that is NOT under any submodule) reverts that one file correctly (sniff: get: graf/DAG.c overwritten from ?), but then runs the submodule pass *unscoped*: be: get abc re-checks-out the ENTIRE abc submodule — every file emits an upd abc/... row (242 objects verified) — even though the requested path graf/DAG.c isn't under abc/. The path filter is applied to the parent checkout but dropped before the submodule recursion. It's content-harmless (the sub is already at its pin d8656f41, so every file is rewritten with identical bytes), but it is wrong and floods the output with a full-tree rewrite for a one-file get. See Submodules, GET, CLAUDE.

*Re-confirmed 2026-06-11* at trunk 97fa76e8 (post-URI-004): be get --force graf/DAG.c in a worktree with a mounted abc sub reverts only graf/DAG.c (sniff: get: graf/DAG.c overwritten from ?) then be: get abc → keeper: verified 242 objects + ~242 upd abc/… rows (sub at pin d8656f41, identical bytes). Still open.

Issues

The path scope of a be get does not propagate to its submodule recursion.

  • A path-scoped be get <path> recurses into and re-checks-out a submodule whose mount the path does not touch — full sub rewrite for a one-file get.
  • The parent honors the path (only graf/DAG.c reverted); the sub pass ignores it and re-materializes every sub file.
  • Harmless on identical content, but a needless full rewrite (mtimes bumped, hundreds of upd rows) — and a real risk if the sub had local edits the path didn't name.

Blockers

None. Localized to the get path-filter → sub-recursion dispatch.

Planned

Scope the sub recursion to the requested path; skip subs the path doesn't reach.

  • Repro-first (CLAUDE §17): be get --force <file-not-under-any-sub> in a worktree with a mounted sub asserts the sub is NOT re-checked-out (no upd <sub>/... rows).
  • When a be get carries a path scope, recurse only into the submodule(s) that scope actually reaches; a path outside every sub mount recurses into none. --nosub must also suppress it.
