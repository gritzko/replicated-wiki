DIS-003: WEAVEMerge token identity is positional, not total

WEAVEMerge aligns weave streams by diff position rather than by real cross-weave token identity, so a small class of merges has a non-total, timestamp-dependent tie-break: a reconciled token can be emitted at the ours position when its true match is a later theirs token, and an insert can anchor on the wrong side of a blank line. The goal is a total, deterministic merge so content recovery holds for every history. The robust fix is a shared-sequence merge giving tokens stable identity, not diff-alignment plus positional reconciliation. See Graf.

Issues

Two faces of one root cause — positional alignment in graf/WEAVE.c WEAVEMerge.

  • Cross-side reconciliation ordering: a byte-equal (a,b) pair reconciled at the ours position loses theirs-relative order when its real match is a later theirs token after a theirs-only token.
  • Fuzz holdout (graf/fuzz/WEAVE2, crash-d924…, in $HOME/Corpus/WEAVE2): 927/928 of the corpus recovers correctly; the lone holdout recovers abac instead of aabc.
  • Insert-vs-blank flake (be-patch-15-ancestor-skip step 2, ~1/2): a re-stamped spine blank collides on (bytes,in) with theirs's code-line EOL, so the LCS anchors the insert on the wrong side of the blank.
  • The flake is two-layered: a non-deterministic ours-closure topo-sort (graf/DAG.c, SHA-keyed) yields either a clean or a duplicate-carrying weave, then WEAVEMerge's full-stream diff mis-ties.

Blockers

A clean fix is constrained.

  • Confined diff-key tweaks (drop-in-for-ws, line-context, blank/EOL bit, alive/dead bit) and an EDL insert-slider stay flaky or regress MERGE3B01 rm_reconciliation_in_disjoint_tail.
  • A total/deterministic closure topo-sort fix lives in graf/DAG.c, which was outside the prior task's edit scope.

Planned

Replace positional reconciliation with real token identity.

  • Rework WEAVEMerge to align on alive-only streams while threading dead tokens to the disjoint-tail rescue separately.
  • Or make the ours-closure topo-sort total/deterministic in graf/DAG.c so equal-timestamp commits sort stably.
  • Keep graf/test/WEAVE2.c content-recovery and MERGE3B01 green; fold the minimized holdout into the property test.

Landed

One sub-bug already fixed.

  • WEAVEMerge in=0 mis-stamp: dedup now keeps BOTH tokens ordered by stamp (earlier marked removed by the later) so every ancestor recovers its own content; pinned by graf/test/WEAVE2.c dedup_merge_post_fork.
