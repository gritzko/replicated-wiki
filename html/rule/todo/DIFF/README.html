DIFF tickets: the diff: view

be diff:<uri> renders the change between versions as a unified/token diff. Tickets here close gaps in what the view shows — submodule recursion, gitlink rendering, scoping. Each entry is one active ticket; it drops off once the whole ticket lands. The fix-and-land method is Issues.

Active tickets

  • DIFF-001 — be diff:?<sha> shows NOTHING for a submodule-pin-bump commit: the gitlink change (old-pin→new-pin) isn't rendered and diff: never recurses into the sub, so a parent commit that advances a sub by real work looks empty. Render the gitlink line and recurse into the sub (like GET/POST/PATCH), --nosub to suppress.
