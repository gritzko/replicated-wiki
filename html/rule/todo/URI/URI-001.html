URI-001: Unify branch/commit naming machinery

*Status: DONE — all 5 stages LANDED / CONFIRMED.* Branch-vs-commit-hash disambiguation WAS scattered across dog / keeper / sniff / graf, each re-deriving ref structure from the string heuristically; the canonical resolved form was unsettled (the DIS-025 pin-in-query shape fought ULOG dedup — see "Canonical form" below); and hash length was hardcoded to 40 (sha1), so nothing was sha256 ready. This ticket unified all of it behind one keeper-owned rule: DOGRefSplitPin is gone, DOGIsHashlet-as-disambiguator is replaced by branch-first GRAFRefIsName / KEEPResolveRef, and the len==40 recognizers route through DOGIsFullSha.

The one rule

Disambiguation lives *only in keeper, is branch-first*, and is *hash-length-agnostic*:

1. Does the ref name a *branch* (REFS lookup)? → branch / trunk. 2. Else, is the ref *all hex* AND does that *hash exist* (pack index)? → hashlet (detached). The index knows sha1 (40) vs sha256 (64); callers never test a length. 3. Else → not found.

*Remote exception:* a ref requested from a peer (ssh:/be:) is NOT disambiguated locally — the raw ref is sent and the remote keeper resolves it.

Canonical form (revised 2026-06-07, LANDED b2b4ff9e)

The earlier DIS-025 plan put the resolved hash in a query *pin* (?/<proj>/<branch>/<sha>, detached ?/<proj>//<sha>). It collides twice: ULOG dedup keys on the row text *before #* (so the tip sha must stay in the REFS fragment — every tip-move of one ref shares a dedup key, dog/ULOG.c:663), and in argv URIs the fragment is already *verb payload* (PATCH squash/merge/ rebase mode, GET #~N/#sha). So the funnel canonicalises the *SCOPE only* and NEVER pins a tip into query or fragment:

  • ?/<proj> — trunk scope.
  • ?/<proj>/<branch> — branch scope (may nest).
  • ?/<proj>/<full-sha> — DETACHED: a bare commit, no branch (branch-first:

the segment is a branch, else the commit).

The resolved sha pin lives ONLY in --at (<root>?/<proj>/<branch>#<sha>, read from the wtlog) and REFS storage (?heads/main#<sha>, UNCHANGED — keeps ULOG dedup intact). Downstream dogs re-resolve the scope to a tip themselves (KEEPResolveRef), so the argv fragment stays free for the verb.

Decisions (2026-06-07): (1) --at's fragment is ALWAYS the resolved sha, NEVER a commit message; the message is a query-less #msg URI (or -m). (2) A future fragment-locator (?/<proj>/<branch>#<hashlet|msg> selecting a SPECIFIC commit in scope, unifying sha / hashlet / message-search) is DEFERRED — today the fragment is left as verb payload.

ZERO on-disk churn: resolver/emitter change only.
