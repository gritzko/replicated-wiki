GET-002: be get/post //host (scheme+authority only) must complete path/query from history

be get ssh://host with only a scheme and authority — no path, no query — should complete the missing parts from the last successful get/post (the persisted clone source), so a bare-host re-fetch resolves the same repo it was cloned from. Instead it fails WIRECLFL (it tries to fetch from the host with an empty repo path). This completion logic existed and worked; it regressed. The goal is to restore URI-completion-from-history for transport args missing path/query. See GET, POST, URI.

Issues

A bare-authority transport arg is sent to the wire as-is instead of being completed from the persisted source.

  • Repro: be get ssh://spot → WIRECLFL / BEDOGEXIT; be get ssh://spot/src/dogs (full path) fetches fine. The clone source (ssh://spot/src/dogs) is recorded — the bare form should reuse its path/query.
  • The completion source is the persisted clone source: the project's line-1 get row / row-0 anchor (the original clone URI), and/or the most recent get/post that carried a host path — the wtlog holds it.
  • Regression: this URI-completion step was present and working; the recent source/anchor/title refactoring (the DIS-001 row-0 / REFSSourceScheme area) likely dropped it. Use git history to locate the removed logic rather than reinventing it.

Blockers

None.

Planned

Restore completion of a transport URI's missing path/query from the persisted source.

  • Repro (rule 17): in a clone whose source is ssh://host/path, be get ssh://host (no path) completes to the full source and fetches; likewise be post //host.
  • AUTHORITY GIVEN (ssh://localhost) — already works: keeper_remote_uri completes the empty path from the matching recorded source (the host-or-path gate). A host that does NOT match the recorded one (ssh://spot) is left alone (a deliberate different remote; use a path).
  • SCHEME ONLY (ssh:, the remaining gap) — find the RECENTMOST get/post row that used that transport scheme and reuse its full URI (authority + path + query). E.g. with a recent ssh://localhost/src/dogs get/post, be get ssh: resolves to it.
  • Add a test/get/ case asserting ssh: (scheme alone) recovers the recentmost same-transport URI (the ssh://host form is covered by test/get/35).
