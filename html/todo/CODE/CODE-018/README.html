CODE-018: manual ok64 propagation should use PRO.h call()/try()

In .c files a recurring hand-rolled pattern ok64 o = A; if (o != OK) return o; forwards an ok64 callee's error WITHOUT a PRO.h frame. Per CLAUDE §1/§16 + ABC, a function that calls a fallible ok64 fn should itself sane()/done and propagate via call(A) (or try(A) + branch). Header-defined fns are inlineables that should be can't-fail; if one does fallible work it belongs in a .c with a real frame. Suspect list (116, regenerable) in suspects. NOT a mechanical sweep — call() snapshots+rewinds ABC_BASS, which manual return o does NOT, so each conversion is a per-case correctness check. Method: Issues.
