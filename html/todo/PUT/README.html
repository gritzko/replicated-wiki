PUT tickets: reflog edits (stage / branch / tag)

be put is the reflog-write verb — stage files (into the .be/wtlog ULOG), set a branch tip, fast-forward a remote. Tickets here cover its reflog-edit correctness. The fix-and-land method is Issues.

Active tickets

  • PUT-002 — *CRITICAL.* be put ?#<sha> can't reset a branch tip: a POST_MIG shared-ancestry walk refuses when the new sha doesn't reach the current tip within POST_MIG_MAX (a re-hashed/partially-indexed history bails), and neither --force nor the ! bang bypasses the gate — contradicting "PUT is unconstrained." A contaminated ?main can't be rolled back through be at all (only hand-editing refs + dropping .refs.idx). A bare/--force tip-set must set the ref OUTRIGHT, no reachability walk.
