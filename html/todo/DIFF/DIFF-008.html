DIFF-008: tree-diff silently truncates later files — per-file BASS scratch is never rewound in graf_diff_tree_refs_inner

The per-file loops in graf_diff_tree_refs_inner call GRAFDiff2Layer with NO call()/try() frame: the ok64 return is IGNORED, and GRAFDiff2Layer's internal BASS scratch is not rewound between files. So scratch accumulates across the file list until a_carve returns NOROOM — and from then on every remaining file's diff SILENTLY FAILS and is dropped from the output. A big-blob file (or just enough files) thus truncates the tail of a be diff:/commit: tree diff with no error. Surfaced by [BLAME-006b] (a 9.5 MB binary starved the last file); its binary-skip MASKED the trigger but did not fix the root. Method: Issues.

Input

Context

  • graf/DIFFREF.c::graf_diff_tree_refs_inner (:513) — the added/changed-file loop

(:555-589) and the deletion loop (:592-612) each end in a BARE GRAFDiff2Layer(path, ext, old_data, new_data, NO, navver); (:588, :611) — the return is discarded, so a per-file NOROOM/failure is swallowed, not propagated.

  • GRAFDiff2Layer (:155) does GRAFArenaInit + the weave carves (e.g. the wt-base

wrapper a_carve(u8, base_buf, 16<<20) :217; WEAVEEmitFull carves the 65536 outtoks etc.). None of that is released between loop iterations — there is no per-file try() rewind or Bu8mark/rewind. The code comment at :370-372 documents the failure mode: "carves 16 MB per file — until a_carve returns NOROOM and the outer call frame unwinds."

  • Symptom (from [BLAME-006b]): on the root-tree diff, the LAST file

(beagle/test/be-unknown-command.sh) was silently truncated because the 9.5 MB dog/dogs.png weave had exhausted the BASS. BLAME-006b's binary-skip removed THAT trigger but a large-enough text set (or many files) still truncates the tail.

Goals

  • Each per-file diff runs inside its OWN BASS frame so its scratch is released before

the next file — scratch is bounded to one file's worth, not the cumulative tree.

  • A per-file GRAFDiff2Layer failure is propagated/surfaced (or at minimum logged),

NEVER silently swallowed into a truncated-but-exit-0 diff.

Constraints

  • Output byte-identical on inputs that fit today; the fix only removes the

accumulation. graf change gates for review. Keep the [BLAME-006b] binary-skip + one-sided short-circuit intact.

WIP

Design decisions

  • (open) Wrap each per-file GRAFDiff2Layer in a try()/then frame (rewinds BASS to

the pre-call mark after each file) — the canonical PRO.h pattern — or take an explicit Bu8mark before the call and rewind after. Stop ignoring the return: call()/try() it so NOROOM/failure propagates instead of truncating.
