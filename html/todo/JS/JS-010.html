JS-010 MED: boundary cleanup — JSStringRef leaks + callback re-entrancy / exception safety

JSValueToStringCopy refs are never released, and the poll callbacks re-enter JS that can mutate JS_FILES/JABC_TIMER mid-loop while a C++ throw unwinding through a C-ABI callback is UB. Method: Issues.

Input

Context

  • (a) JSValueToStringCopy returns a +1 ref never released — utf8.cpp:18

(JABCutf8Encode), convert.cpp:77.

  • (b) Poll callbacks (timeout_cb io.cpp:176-189, JABCioNetOnEvent

io.cpp:281-336) re-enter JS that can mutate JS_FILES / JABC_TIMER mid-loop with no guard; a C++ throw unwinding through these C-ABI callbacks is UB.

  • WITH_JS-gated WIP; convert.cpp is the dead rdx path, utf8.cpp is live.

Goals

  • Every JSStringRef released; callbacks survive re-entrant JS_FILES/JABC_TIMER

mutation; no C++ exception escapes a C-ABI callback.

Constraints

  • Pair every JSValueToStringCopy with JSStringRelease; wrap callback bodies in

try{}catch(...){} (effectively noexcept); document the single-thread contract and add a re-entrancy assert.

WIP

Design decisions

  • (open) Add the release at each site; make the callbacks exception-safe and

re-entrancy-aware.
