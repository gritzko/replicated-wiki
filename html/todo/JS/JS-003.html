JS-003 HIGH: JSC boundary not validated before use — ignored exception, NULL typed-array bytes ptr

JS* calls set an exception out-param that is ignored before the code proceeds to the syscall, and a detached ArrayBuffer's NULL bytes ptr is deref'd. Flagged by 3-4 of 7 lenses (high confidence). Method: Issues.

Input

Context

  • (a) exception out-param ignored, then the syscall runs anyway:

JABCioFileWrite (io.cpp:97-111), JABCioFileRead (io.cpp:137-160), JABCioNetConnect (io.cpp:374-376), JABCioStdio (io.cpp:385-392), JABCutf8Encode (utf8.cpp:18).

  • (b) JSObjectGetTypedArrayBytesPtr returns NULL for a detached/neutered

ArrayBuffer; io.cpp:100-105 / 137-141 form ta[1]=ta[0]+len off NULL and read/write-deref it → NULL-deref crash.

  • WITH_JS-gated WIP, but these are live binding entry points.

Goals

  • A thrown JS exception or a detached buffer is detected and aborts the op before

any syscall / deref.

Constraints

  • After each JS* call: if (*exception) return undefined. NULL-check the bytes

ptr before forming the ABC slice. No bypassing the existing throw plumbing.

WIP

Design decisions

  • (open) Add post-call exception checks at every listed site; guard the bytes ptr

so a detached buffer throws (or returns undefined) rather than deref'ing NULL.
