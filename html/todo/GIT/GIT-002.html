GIT-002: object-by-object OFS-only writer — one growing log per shard, never REF_DELTA

Input

Context

Child of [GIT-001]. The store's two write paths diverge: local commits go through KEEPPackFeed (keeper/KEEP.c:1652) which emits OFS_DELTA only when the base is raw in the same pack (keep_find_raw_in_pack) and falls back to REF_DELTA otherwise (:1724); ingest (KEEPIngestFile:2238) u8bFeeds a whole foreign pack verbatim. Converge both onto ONE per-object append — a dog/git format function that writes raw objects + OFS_DELTA exclusively, base anywhere earlier in the log, addressed purely by offset — so keeper's KEEPPackFeed, the GIT-003 ingest funnel, AND the js/JABC binding all call one writer (the binding holds no pack-log logic). This is the foundational change; everything else in [GIT-001] builds on it. See PackLog, CLAUDE.

  • Dual writers: KEEPPackFeed (Feed-time delta decision) and the ingest verbatim append are separate code with separate delta policy; a format change must be written twice and they can drift.
  • REF_DELTA on write: dtype = in_pack ? PACK_OBJ_OFS_DELTA : PACK_OBJ_REF_DELTA (KEEP.c:1724) is the single line that injects sha-addressed deltas into storage — the source of the cross-file recursion and droppability hazards [GIT-001] removes.
  • OFS candidate is pack-batch-scoped: keep_find_raw_in_pack only scans the in-progress pack's entries, so cross-batch near-identical objects re-store raw; one growing log lets OFS reference any earlier object in the shard.

Goals

  • One per-object append writer in dog/git that emits raw objects + OFS_DELTA exclusively (never REF_DELTA), base any earlier object in the open shard log, addressed purely by offset.
  • Converge keeper's KEEPPackFeed, the GIT-003 ingest funnel, and the GIT-007 JABC binding onto that single append code path (the binding holds no pack-log logic).

Constraints

  • OFS varint is the only thing rewritten — keep the deflated delta-instruction bytes; recompute the header offset as cur_pos - base_pos. No re-deltification, no re-deflate on the verbatim-base path. Use dog/git/PACK.h PACKu8sFeedObjHdr + the OFS varint encoder; no open-coded headers (CLAUDE §10, no ptr arithmetic).
  • Preserve the intra-pack type order invariant (PackLog §"Intra-pack object order"): non-decreasing commit→tree→blob→tag for canonical writes; strict_order stays for Feed, ingest emits in dependency order (GIT-003).
  • Layering: the byte-level append + the OFS varint encoder live in dog/git (beside PACKu8sFeedObjHdr), taking a caller-provided log buffer + resolved base bytes/offset — no refs, index, or singleton. keeper keeps the policy (KEEPObjSha, base resolution via keep_find_raw_in_pack/KEEPGet, WHIFFHashlet60, indexing, the FILEBook growable log); reuse DELTEncode/ZINFDeflate, no fresh re-rolls.
  • Delete the duplicated OFS encoder: keeper/KEEP.c:1544 pack_feed_ofs and js/pack.hpp:23 JABCPackFeedOfs are two hand-rolled copies — replace both with one PACKu8sFeedOfs in dog/git (the missing encode counterpart to PACKDrainOfs).
  • Base must precede dependent (forward-reference free) — the writer only ever deltas against an already-appended object.
