GIT-005: single-stream serving — verbatim whole-log clone + a thin-pack builder for incremental fetch

Input

Context

Child of [GIT-001]; depends on [GIT-002]. With one OFS-only stream per shard, full-clone serving is trivial: the whole log IS a valid git pack sans framing, so blast it verbatim with a fresh PACK header + SHA-1 trailer (PSTRWrite, the existing stitcher). The hard case is the incremental fetch: a byte suffix from the client's have-frontier is NOT self-contained, because OFS_DELTA records near the tail can reference bases before the frontier. The current WIREBuildSegments (keeper/WIRE.c:319) tiles PACK bookmarks to ship a verbatim range — that relied on self-contained per-fetch batches, which one-stream dissolves (and whose count math caused [GET-019]). Add a real thin-pack builder that walks the wanted set and thinnifies the boundary. See PackLog, Keeper.

  • Suffix isn't shippable verbatim: a tail OFS_DELTA whose base sits before the watermark can't resolve in a non-thin suffix pack, and git OFS can't reference outside the pack.
  • Bookmark tiling is going away: GIT-003 drops the per-pack (count, byte_len) bookmark, so wire_tile_count / wire_find_pack lose their unit; the count-vs-bytes mismatch class ([GET-019]) goes with it.
  • Two regimes, one server: full clone (no haves) wants raw verbatim throughput; incremental (haves present) wants a minimal thin pack — the serve path must pick.

Goals

  • Verbatim whole-log full-clone serving via PSTRWrite when the client has no haves.
  • A real thin-pack builder for incremental fetch that walks the wanted set and thinnifies the boundary, replacing the bookmark-tiling arm of WIREBuildSegments.

Constraints

  • Storage stays untouched: the builder is a serve-time transform (reads the OFS-only log, emits a wire pack); it never rewrites the log. Mirrors the PackLog "cheaply reframed" property, extended to thin output.
  • Wire-correct thin packs: bases the client already has (its haves' reachable closure) become REF_DELTA; bases inside the shipped set stay OFS_DELTA with offsets recomputed for the OUTPUT pack. Negotiate ofs-delta cap as today (WIRE_CAP_OFS_DELTA).
  • Keep PSTRWrite as the verbatim stitcher for the full-clone path; reuse dog/git/PACK.h header/varint emit and DELTEncode — no open-coded pack bytes (CLAUDE §10).
  • Both wire ends: serve git clients (vanilla git fetch) AND beagle be:// peers; a beagle peer could accept the raw OFS-only stream directly, but emit standard packs for uniformity unless a measured win says otherwise.
