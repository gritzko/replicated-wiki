POST tickets: commit / fast-forward / push

be post advances the current branch — worktree→repo commit and/or fast-forward push, recursing into mounted submodules. Tickets below tighten its commit-message handling, push transports, and no-op reporting. Each entry is one active ticket with its live state; an entry drops off once the whole ticket lands (landed commits stay on the board's POST line). The fix-and-land method is Issues.

Active tickets

  • POST-006 — POSTCFLCT false-fires on conflict-marker strings in tracked source comments/literals (e.g. sniff/PATCH.c:339), forcing a blunt --force. Recognize only the real WEAVE output shape, not any inline <<<<…||||…>>>> occurrence.
  • POST-007 — be post mangles be://→be:/ in commit messages (a //-collapsing path normalization) and rejects multi-line / #-led messages as path-form URIs. Treat the message slot as opaque text that round-trips byte-identical.
  • POST-013 — a submodule on the synthetic ?.beagle/<parent> branch refuses to push to a git remote without explicit ?main. When the local branch is a be-only synthetic coordinate and the target is git, default to the remote's advertised HEAD / main / master.
